#  Supermarket Database Management System (MySQL Project)

##  Overview
This project is a complete **end-to-end MySQL database system** designed for managing a supermarket’s customers, products, categories, orders, order details, and analytical reporting.

It includes:
-  Database creation  
-  Table creation with primary & foreign keys  
-  Sample data insertion  
-  Business SQL queries  
-  Window functions (RANK, ROW_NUMBER, LAG, LEAD, etc.)  
-  SQL Views  
-  Stored Procedures for CRUD operations  
-  Real-world analytical queries  

---

## Project Features

### Database & Table Design
Includes creation of:
- Customers table  
- Products table  
- Categories table  
- Orders table  
- Order Details table  

All tables are linked using **Primary Key** and **Foreign Key** relationships.

---

## Business SQL Queries  
This project includes many real-world business analytics queries such as:
- Filtering customer information  
- Joining tables for detailed order info  
- Category-wise product lists  
- Total revenue per product  
- Top-selling product analysis  
- Customers who never placed an order  
- Detailed customer order history reports  

---

##  SQL Views  
Pre-built SQL views for simplified analysis:
- **TopSellingProducts**  
- **HighestSellingProductWithCustomer**

---

##  Stored Procedures  
This project includes multiple automated stored procedures:
- GetCustomersByCountry  
- GetOrdersByCustomer  
- InsertProduct  
- GetOrderDetailsByOrderID  
- GetTotalSalesByProduct  
- UpdateOrderStatus  
- DeleteOrder  
- GetMostOrderedProducts

##   Business Questions  
This SQL project answers real business problems such as:

- Which customers are from Germany?  
- What are the top 5 most sold products?  
- What is the total revenue for each product?  
- Which orders were refunded?  
- Which customers have never placed an order?  
- Rank products by price within categories  
- Show next and previous order amounts for each customer
